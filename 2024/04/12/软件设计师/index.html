操作系统的分类

分类 特点
批处理操作系统 单道批：一次一个作业入内存，作业由程序、数据、作业说明书等组成
多道批：一次多个作业入内存，特点：多道、宏观上并行，微观上串行
分时系统 采用时间片轮转的方式为多个用户提供服务，每个用户感觉独占系统
特点：多路性、独立性、交互性和及时性
实时操作系统 实时控制系统和实时信息系统
交互能力要求不高，可靠性要求高（规定时间内响应并处理）
网络操作系统 方便有效共享网络资源。提供服务软件和有关协议的集合
主要的网络操作系统由：Unix、Linux、Windows Server
分布式操作系统 任意两台计算机可以通过通信交换信息
是网络操作系统的更高级形式，具有透明性、可靠性和高性能等特性
危机操作系统 Windows: Microsoft开发的图形用户界面、多任务、多线程操作系统
Linux: 免费使用和自由传播的类Unix操作系统，多用户、多任务、多线程和多CPU的操作系统
嵌入式操作系统 运行再智能芯片环境中
特点：微型化、可定制（针对硬件变化配置）、实时性、可靠性、易移植性（HAL和BSP支持）

进程管理

进程是程序再一个数据集合上运行的过程，它是系统进行资源分配和调度的一个独立单位，它由程序块、进程控制块（PBC）和数据块三部分组成。

进程与线程的概念

  • 进程与程序的区别：进程是程序的一次执行过程，没有程序就没有进程
  • 程序是一个静态的概念，而进程是一个动态的概念，它由创建而生产，完成任务后因撤销而消亡
  • 进程是系统进行资源分配和调度的独立单位，而程序不是

进程的两个基本属性：可拥有资源的独立单位；可独立调度和分配资源的基本单位

拥有资源的独立单位 独立调度和分配资源的基本单位
线程、程序计数器、寄存器栈 内存地址空间、代码、数据、文件等

进程的状态

状态 详情
运行 当一个进程在CPU上运行时。（单处理机处于运行状态的进程只有一个）
就绪 一个进程获得了出CPU外的一切所需资源，一旦得到处理机即可运行
阻塞(等待) 一个进程正在等待某一事件发生(例如请求I/O、等待I/O完成等)而暂时停止运行，此时即使把CPU分配给进程也无法运行，顾称进程处于阻塞状态

信号量与PV操作

P 要资源 V 释放资源

临界资源： 诸进程间需要互斥方式对其进行共享的资源。

信号量： 是一种特殊的变量（可以表示资源数量；为负数时还可以表示排队的进程数）

模型 特点
互斥模型 互斥信号量S的初始值一般为非0
访问权是一类特殊的互斥资源，同一时刻仅允许一个人用，则信号量初始值为1
先P后V
同步模型 同步信号量S的初始值一般为0
先V后P

前驱图与PV操作

  • 并发图中某活动有后继就有V操作释放资源，有前驱就有P操作消耗资源

  • 实现并发的信号量初始值一般为0、有几个箭头就有几个信号量

死锁问题

所谓死锁，是指两个以上的进程互相都要求对方已经占有的资源导致无法继续运行下去的现象。

条件 死锁处理 详情
1、互斥 死锁的预防 (打破四大条件)
2、保持和等待 死锁的避免
3、不剥夺 死锁的检测与接触
4、环路等待 鸵鸟策略（不予理睬）

系统不可能发生死锁的最小资源数： (w-1)*m+1 <= n

w: 所需要的资源

m: 进程数
n：总资源

存储管理器

段页式存储

高级程序语言使用逻辑地址

运行状态，内存中使用物理地址

类别 优点 缺点
页式存储：将程序与内存均划分同样的大小块，以页为单位将程序调入内存 利用率高，碎片小，分配及管理简单 增加了系统开销；可能产生抖动现象
段式存储：按用户作业中的自然段来划分逻辑空间，然后调入内存，段的长度可以不一样 多道程序共享内存，各段程序修改互不影响 内存利用率低，内存碎片浪费大
段页式存储：段式与页式的综合体。先分段，再分页。
一个程序有若干段，每个段中可以有若干页，每个页的大小相同，但每个段的大小不同		 空间浪费小、存储共享容易、存储保护容易、能动态连接 由于管理软件的增加，复杂性和开销页随之增加。需要的硬件以及占用的内容页有所增家，使得执行速度大大下降

考点：

  • 页式存储：

    逻辑地址=页号+页内地址

    物理地址=页帧号+页内地址

  • 段式存储

    逻辑段地址:判断偏移量是否合法（短号，偏移量）

  • 段页式存储

    根据地址结构计算相关数据

  • 找磁道的时间
  • 找扇区的时间
  • 传输时间

移臂调度算法：先找磁道，再找扇区。

目标：使用磁盘平均寻道时间最少

  • 先来先服务(FCFS)

  • 最短寻道时间优先(SSTF)

  • 扫描算法(SCAN)

  • 循环扫描(CSCAN)算法

第二章 算法

算法的效率

时间复杂度：是指程序运行从开始到结束所需要的时间。

空间复杂度：是指对一个算法在运行过程中临时存储空间大小的度量。

常见的对算法执行所需时间的度量：

O(1) < O(log₂n) < O(n) < O(nlog₂n) < O(n²) < O(n³) < O(2ⁿ) < n!

单个语句O(1)<二分查找O(log₂n)<单层循环<单层循环O(n)<双层循环O(n²)<堆排序O(nlog₂n) <判断是否包含指定序列O(2ⁿ)

查找

名称 考点
顺序查找 顺序查找的平均查找长度为(n+1)/2
二分查找 折半查找在查找成功时关键字的比较次数最多为log₂n+1
折半查找的时间复杂度为O(log₂n)次
折半查找的前提：有序、顺序存储
哈希查找(散列表查找) 哈希冲突

排序

类别 排序方法 平均时间复杂度 特使情况时间复杂度 空间复杂度 稳定性
插入排序 直接插入 O(n²) 基本有序最优O(n) O(1) 稳定
插入排序 Shell插入 O(n^1.3) 基本有序最优O(n) O(1) 不稳定
选择排序 直接选择 O(n²) 基本有序最优O(n) O(1) 不稳定
选择排序 堆排序 O(nlog₂n) 基本有序最优O(n) O(1) 不稳定
交换排序 冒泡排序 O(n²) 基本有序最优O(n) O(1) 稳定
交换排序 快速排序 O(nlog₂n) 基本有序最差O(n²) O(1) 不稳定
归并排序 归并排序 O(nlog₂n) 基本有序最差O(n²) O(n) 稳定
基数排序 基数排序 O(d+(b+rd)) 基本有序最差O(n²) O(rd) 稳定
  • 若待排序列的计数目n较小，可采用直接插入排序和简单选择排序。
  • 若待排序记录按关键字基本有序，宜采用直接插入排序或冒泡排序。
  • 当n很大且关键字位数较少时，采用基数排序较好。
  • 如n很大，则采用时间复杂度为O(nlog₂n)的排序方法。
    1. 快速排序目前被认为是内部排序最好的方法，当排序的关键字为随机分布时，快速排序的平均运行时间最短。
    2. 堆排序只需要一个辅助空间，并且不会出现在快速排序中可能出现的最快情况。
    3. 快速排序和堆排序都是不稳定的排序方法，若要求排序稳定，可选择归并排序。

算法策略

算法策略 特征 经典问题
分治法 把一个问题拆分为多个小规模的相同子问题，一般可用递归解决 斐波那契数、归并排序、二分搜索、最大数乘法、汉诺塔
贪心法 局部最优，但整体不见得最优。每步有明确的、既定的策略。 背包问题、找零问题、最小生成树
动态规划 使用数组存储子问题结果，利用查询子问题结果构造最终问题结果。 斐波那契数列、矩阵乘法、背包问题
回溯发 系统地搜索一个问题的所有解或任一解 N皇后问题、迷宫、背包问题
